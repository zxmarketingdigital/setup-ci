Criar e atualizar Setups do ZX Control sem saber programar

Você conversa com o Claude em português. Ele faz todo o trabalho técnico — git, terminal, código. Você nunca digita comando e não consegue quebrar nada.

🔒 Tudo que você faz vira um “pedido de aprovação”. Nada vai pro ar sem o Rafael revisar.
Configuração inicial

2

3
Quando pedir, clique em conectar o GitHub e autorize. (É onde os setups ficam guardados.)

Fluxo principal

Atualizar um Setup que JÁ existe

Mais simples ainda — não precisa criar nada.

1
VocêNo claude.ai/code, abra o repositório do setup que quer mexer.
2
VocêDiga o que mudar:
Neste setup, corrige / adiciona [o que for]. Leia o CLAUDE.md antes.
3
Vocêvalida o setup → abre o Pull Request → avise o Rafael.
Como funciona por trás

Você não precisa decorar regras

🤖 O Claude já sabe as regras

As 12 regras do ZX Control vivem dentro do repositório. O Claude lê sozinho e segue.

✅ A validação confere tudo

Ao abrir o Pull Request, um “fiscal” automático checa as 12 regras. Se algo estiver errado, aparece vermelho.

🔒 Nada vai pro ar sem aprovação

Você só propõe. O Rafael é o único que aprova e publica. Impossível quebrar a produção.

🙅 Você nunca usa o terminal

Git, comandos, deploy — tudo é o Claude. Você só conversa em português.

Perguntas comuns

FAQ

Preciso digitar comandos no terminal?
Não. Você só conversa com o Claude. Ele cuida do terminal, do git e de tudo.
E se a validação reclamar de alguma coisa?
Peça pro Claude corrigir: “corrige os erros que a validação apontou”. Ele resolve e valida de novo.
Posso quebrar a produção ou os alunos?
Não. Você só abre um pedido (Pull Request). Nada entra no ar até o Rafael aprovar.
O que é um “Pull Request”?
É um pedido de revisão: você propõe uma mudança e o Rafael decide se entra. Como mandar um documento pra aprovação.
Travei e não sei o que fazer.
Escreva pro Claude: “estou perdido, me explica o próximo passo de forma simples”. Se ainda travar, chame o Rafael.
